## bench-action@1.0.0

First release — a compiled `node20` GitHub Action that turns
[`@soroush.tech/bench`](https://www.npmjs.com/package/@soroush.tech/bench)
into a CI performance gate. One `uses:` step, zero setup: the bundled runner
(`dist/index.cjs`) and the vendored sandbox (`vendor/Dockerfile` +
`vendor/dist/harness.mjs`) are committed, and GitHub-hosted Linux runners
already ship Docker.

### Features

- Runs every `*.bench.ts` file in a caller-supplied `bench-dir` (or a `files`
  subset) inside the CPU/RAM-pinned Docker sandbox — each file's cases are
  compared against the `baseline-case`, and any case whose speed ratio falls
  below `min-ratio` (percent) fails the file and, with it, the job.
- Posts the results as a single sticky PR comment (hidden-marker matched, so
  repeat pushes update it in place instead of stacking new ones). On a
  read-only token (e.g. a fork PR) the comment is skipped with a warning —
  only the gate decides the job.
- Sandbox pass-throughs (`rounds`, `cpus`, `cpuset`, `memory`) for tuning
  noise-resistance on shared runners; `rounds` gates on the median per case.
- `report` output carries the combined markdown report for downstream steps.
- The results directory is world-writable before mounting, so the sandbox's
  unprivileged user can write the per-file table on native Linux Docker.

### Consuming

```yaml
- uses: soroush-tech/bench-action@v1
  with:
    bench-dir: bench
    baseline-case: previous
    min-ratio: '80'
    rounds: '5'
    github-token: ${{ secrets.GITHUB_TOKEN }}
```
