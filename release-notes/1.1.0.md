## bench-action@1.1.0

The results comment now works with zero configuration and can carry the bench brand:
`github-token` defaults to the workflow's automatic runtime token, and when the public
bench GitHub App is installed on the repo (plus `id-token: write` on the job) the comment
is posted **as the bench bot** through the hosted relay at `api.bench.soroush.tech`
(soroush-tech/core#308).

### Features

- **Branded comments**: with the bench GitHub App installed and `id-token: write`
  granted, the sticky PR comment is authored by the bench bot via the OIDC-verified
  comment relay. Any relay failure (app not installed, no OIDC permission, relay down)
  logs the reason and falls back to the direct comment — the gate verdict is never
  affected. Opt out with `branded: 'false'`.
- `github-token` defaults to `${{ github.token }}`, so the fallback comment needs no
  setup either. Pass a GitHub App token of your own to brand the fallback, or an
  explicit `''` to skip commenting; off-PR runs and read-only tokens still skip the
  comment gracefully.

### Changed defaults

- Previously, omitting `github-token` skipped the comment. To keep that behavior, set
  `github-token: ''` and `branded: 'false'` explicitly.
- A pre-existing sticky comment keeps its original author when the relay updates it —
  delete the old bench comment once to rebrand from the next run.

### Internal

- `dist/` and `vendor/` are now produced at release time by the Release workflow;
  `main` stays TypeScript-only (#5, #6).
