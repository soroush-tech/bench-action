  ## bench-action@1.0.1

Internal release-plumbing change — nothing in the action's inputs, outputs,
or behavior changes for consumers. The vendored sandbox (`vendor/Dockerfile`
+ `vendor/dist/harness.mjs`) is no longer committed to `main`: it is now
regenerated at release time from the pinned
[`@soroush.tech/bench@2.1.0`](https://www.npmjs.com/package/@soroush.tech/bench)
npm release and shipped on the release tag, exactly like the `dist/` bundle.
`main` is now fully TypeScript-only.

### Fixes

- The vendored harness now comes verbatim from the published bench `2.1.0`
  npm artifact instead of a monorepo snapshot, eliminating any drift between
  the two builds.
